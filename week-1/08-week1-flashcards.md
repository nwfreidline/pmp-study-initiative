Review these daily during Week 1. Cover the "Answer" column and test yourself.

## Exam Structure

| Term | Answer |
|------|--------|
| How many questions on the PMP exam? | 180 (175 scored + 5 pretest) |
| How much time for the exam? | 230 minutes (3 hrs 50 min) |
| When are the breaks? | After Q60 and Q120 (10 min each, optional) |
| What are the 3 exam domains? | People (42%), Process (50%), Business Environment (8%) |
| What scoring levels does PMI use? | Above Target, Target, Below Target, Needs Improvement |
| What is the ECO? | Exam Content Outline — the blueprint for what the exam tests |
| How many tasks in the ECO? | 35 (14 People + 17 Process + 4 Business Environment) |

## Delivery Approaches

| Term | Answer |
|------|--------|
| Predictive (Waterfall) | Sequential, plan-driven, upfront requirements, formal change control |
| Adaptive (Agile) | Iterative, incremental, evolving requirements, backlog reprioritization |
| Hybrid | Combination of predictive and adaptive elements |
| When to use Predictive? | Stable requirements, regulated environment, fixed-price contract |
| When to use Agile? | Unclear/changing requirements, high customer involvement, innovation |
| When to use Hybrid? | Mix of stable and uncertain, organization transitioning, both governance styles needed |

## PMI Mindset

| Rule | Answer |
|------|--------|
| Default leadership style? | Servant leadership |
| When a change is requested? | Assess impact → submit change request → CCB reviews |
| When a problem arises? | Assess/analyze BEFORE taking action |
| Gold plating? | NEVER add unrequested scope |
| Bad news? | Communicate proactively and honestly |
| Stakeholder engagement? | Early and continuous, not one-time |
| Ethical dilemma? | Choose the ethical path — report, disclose, refuse |
| Project charter? | Authorizes the project and the PM — issued by sponsor |
| Risk management? | Proactive, not reactive |
| Documentation? | Always document — lessons learned, decisions, rationale |

## Key Distinctions

| Concept A | vs | Concept B | Difference |
|-----------|-----|-----------|-----------|
| 3 Exam Domains | vs | 8 Performance Domains | Exam structure vs PMBOK 7 framework |
| Risk | vs | Issue | Potential future event vs current problem |
| QA | vs | QC | Process audits vs deliverable inspection |
| Leading indicator | vs | Lagging indicator | Predicts future vs measures past |
| Crashing | vs | Fast-tracking | Add resources (cost↑) vs parallel work (risk↑) |
| Gold plating | vs | Scope creep | Team adds unrequested features vs unauthorized scope growth |
| Leader | vs | Manager | Influence and inspire vs plan and organize |
| Sprint Review | vs | Retrospective | Demo product to stakeholders vs team process improvement |
| Mentoring | vs | Coaching | Long-term development vs specific skill/task |
